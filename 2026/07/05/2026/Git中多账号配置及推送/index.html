Git中多账号配置及推送

发布日期:   2026-07-05
更新日期:   2026-07-07
文章字数:   992
阅读时长:   4 分
阅读次数:  

一、核心原理

Git 的用户名、邮箱分两套：

  1. 全局配置：所有仓库默认共用一套 user.name / user.email

  2. 本地仓库配置：单独给某一个仓库设置专属账号信息，优先级高于全局

两个 GitHub 账号对应两套不同的 name+注册邮箱，只要每个仓库单独配置本地 user 信息，提交、推送就不会出现账号混淆、贡献图不显示、权限报错问题。

二、解决方法

方式 1：HTTPS 克隆仓库

  • HTTPS 会缓存账号密码 / PAT 令牌，切换仓库容易推送到另一个账号时报 403。

  • 解决方案：

    1. 清除系统保存的 Git 凭据

      Win：控制面板 → 用户账户 → 凭据管理器 → Windows 凭据，删除 github.com 相关记录

    2. 每次推送对应仓库时，弹窗输入当前仓库所属账号的 PAT 令牌代替密码

方式 2：SSH 克隆

  1. 设置账户和邮箱

    • 进入仓库 A 根目录，打开 Git bash：

      设置账号A的用户名（GitHub主页显示的用户名）

git config user.name "lvchungui"

设置账号A注册GitHub的邮箱

git config user.email "617726681@qq.com"

    • 进入仓库 B 根目录，终端执行：

      git config user.name "bjxstpl"

git config user.email "3604874561@qq.com"

    • 验证是否设置成功（每个仓库单独执行查看）：

      git config --list
      能看到当前仓库独有的 `user.name`、`user.email` 即生效
也可在`.git`文件夹下查看`config`文件是否设置成功

  2. 生成两套 SSH key，分别绑定两个 GitHub 账号

    • 不用进仓库执行密钥生成命令，任意终端窗口都可以

    • 账号 A：生成 id_rsa_a

      ssh-keygen -t rsa -C "617726681@qq.com"

一路回车 → 在.ssh文件夹中重命名生成的2个文件

    • 账号 B：生成 id_rsa_b

      ssh-keygen -t rsa -C "3604874561@qq.com"

一路回车 → 在.ssh文件夹中重命名生成的2个文件

  3. 在 .ssh 文件夹新建 config 文件，写入配置

    # GitHub账号A
Host github-lvchungui
HostName github.com
IdentityFile ~/.ssh/id_lcg
User git

# GitHub账号B
Host github-bjxstpl
HostName github.com
IdentityFile ~/.ssh/id_bjx
User git

  4. 修改仓库远程地址

    • 进入.git文件夹下的config文件，修改url地址

      仓库 A 远程地址改成：git@github-a:用户名A/仓库名.git

git@github-lvchungui:lvchungui/lvchungui.github.io.git
      仓库 B 远程地址改成：git@github-b:用户名B/仓库名.git

git@github-bjxstpl:bjxstpl/bjxstpl.github.io.git

    • 也可以进入仓库文件夹后用命令修改

      git remote set-url origin git@github-lvchungui:lvchungui/lvchungui.github.io.git

git remote set-url origin git@github-bjxstpl:bjxstpl/bjxstpl.github.io.git

  5. 在 Github 中添加秘钥

    • 用记事本打开 id_xxx.pub 将全部的内容复制
    • 打开 Github →进入设置→进入ssh设置→点击添加秘钥，然后将刚刚复制的粘贴进去就可以了

  6. 验证是否配置成功：在任意位置打开git bash分别输入下面命令，看到Hi，xxx即配置成功：

    ssh -T git@github-bjxstpl

ssh -T git@github-lvchungui

  7. ssh克隆优势：两套密钥隔离，推送自动匹配对应账号，完全不会权限冲突，长期开发首选

三、常见报错解决方法

  1. git push出现报错：ssh: connect to host github.com port 22: Connection refused
    • 原因：开了github加速访问之类的工具
    • 解决方法：先停止工具服务，再关闭工具
  2. TortoiseGit push出现报错：No supported authentication methods available(server sent: publickey)

    • 原因：“ 没有支持的认证方法 ”，我们需要修改 TortoiseGit 网络的 SSＨ 客户端为 git 服务器的 ssh.exe

    • 解决方法：

      • TortoiseGit 使用的是自己的 TortoiseGitPlink.exe 作为 SSH 客户端，鼠标右键 → TortoiseGit → Settings → Network → SSH Client 项，如下图：

    • 点击 “ Browser ” 找到在本地安装的 git 服务的根目录 → Git → usr → bin 目录下选中 ssh.exe ，再点击打开即可，如下图：

    • 回到客户端后，点击 “ 应用 ” 即可
